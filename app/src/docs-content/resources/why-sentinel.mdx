# Why Sentinel

## The Gap

x402 enables internet-native micropayments. AI agents can pay for API calls in real time. But the protocol is payment-focused — it doesn't address the operational and compliance needs of enterprises running agent fleets.

## What's Missing

| Need | x402 | Sentinel |
|------|------|----------|
| Audit trail | No | Yes — every payment logged |
| Budget limits | No | Yes — per-call, hourly, daily, lifetime |
| Attribution | No | Yes — agent, team, sponsor |
| Spike detection | No | Yes — N× rolling average |
| Endpoint control | No | Yes — allowlist/blocklist |
| Export for compliance | No | Yes — CSV/JSON |
| Approval workflows | No | Yes — human-in-the-loop above threshold |

## Enterprise Blockers

Surveys indicate compliance is a top blocker for autonomous agent payments:

- **Visibility**: Which agent spent how much on what?
- **Control**: How do we cap spend before it happens?
- **Audit**: How do we export records for auditors?

Sentinel addresses all three with a single line of code.

## Design Principles

- **Drop-in**: Same fetch interface. Remove Sentinel, code works identically.
- **Local-first**: No required remote API. File storage works offline.
- **Non-fatal audit**: Audit failures never block payments.
- **Pluggable**: Swap storage backends, add hooks, customize policies.
