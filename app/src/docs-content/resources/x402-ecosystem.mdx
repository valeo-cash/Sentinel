# x402 Ecosystem

Sentinel builds on the [x402 protocol](https://github.com/coinbase/x402) for HTTP-based payments.

## What is x402?

x402 extends HTTP with a payment flow:

1. Client requests a resource
2. Server returns `402 Payment Required` with payment requirements
3. Client pays (e.g., on-chain via EVM)
4. Client retries with payment proof
5. Server returns the resource

The protocol is scheme-agnostic. Common schemes include `exact` (pay exact amount) and `streaming` (pay-as-you-go).

## Key Packages

| Package | Role |
|---------|------|
| `@x402/core` | Protocol types and utilities |
| `@x402/fetch` | `wrapFetchWithPayment` — fetch wrapper that handles 402 |
| `@x402/evm` | EVM/Base chain payment schemes |

## Where Sentinel Fits

```
Your Agent
    ↓
wrapWithSentinel (Sentinel)
    ↓
wrapFetchWithPayment (x402)
    ↓
fetch (native)
```

Sentinel wraps the x402 fetch. It intercepts payment events (before and after) to enforce budgets and log audit records. It does not modify the payment flow itself.

## Compatible Clients

Sentinel works with any x402-compatible fetch:

- `@x402/fetch` + `x402Client`
- Legacy `x402-fetch` + viem
- Any function with the fetch signature that handles 402 internally
