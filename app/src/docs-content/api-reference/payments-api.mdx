# Payments API

Query payment and audit records from the remote API. These endpoints are used by the dashboard when configured with `apiKey` and `baseUrl` (future integration).

## List Payments

```
GET /v1/payments
```

**Query Parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `agentId` | string | Filter by agent |
| `team` | string | Filter by team |
| `startTime` | number | Unix ms |
| `endTime` | number | Unix ms |
| `status` | string | `allowed`, `flagged`, `blocked` (comma-separated) |
| `limit` | number | Max results (default 100) |
| `offset` | number | Pagination offset |

**Response:**

```json
{
  "records": [
    {
      "id": "a1b2c3",
      "agent_id": "agent-1",
      "amount": "0.50",
      "endpoint": "https://api.example.com/paid",
      "tx_hash": "0x...",
      "policy_evaluation": "allowed",
      "created_at": 1700000000000
    }
  ],
  "total": 42
}
```

## Get Payment by ID

```
GET /v1/payments/:id
```

**Response:** Single `AuditRecord` or 404.

## Current Status

The Payments API is part of the planned api.valeo.money backend. The SDK's `ApiStorage` and `SentinelDashboard` currently run queries **locally** against the configured storage. Remote query endpoints will be documented when available.
