# Introduction

## What is Sentinel?

Sentinel is an enterprise audit and compliance layer for [x402](https://github.com/coinbase/x402) payments. It wraps your existing x402-enabled fetch with budget enforcement, audit logging, and anomaly detection — without changing your API.

## The Problem

The x402 protocol enables internet-native micropayments. AI agents can pay for API calls, LLM tokens, and compute in real time. But x402 provides no built-in:

- **Audit trail** — Who spent what, when, and where?
- **Budget controls** — How do you cap spend per agent, per hour, per day?
- **Compliance tooling** — How do you export records for auditors?

Enterprise teams need answers before they can trust autonomous agent payments in production.

## Why x402 Needs Audit Infrastructure

x402 is a payment protocol. It handles the mechanics of paying and settling. It does not handle:

- **Attribution** — Linking payments to specific agents, teams, or cost centers
- **Policy** — Enforcing limits before payment happens
- **Observability** — Querying spend patterns, detecting anomalies, exporting for compliance

Sentinel fills this gap. It sits between your agent code and the x402 client, intercepting payment events to log, evaluate, and optionally block them.

## Key Features

| Feature | Description |
|---------|-------------|
| **Budget Enforcement** | Per-call, hourly, daily, lifetime limits. Blocks before payment. |
| **Spike Detection** | Flags payments that exceed N× the rolling average |
| **Audit Trails** | Every payment logged with agent ID, team, endpoint, tx hash |
| **Endpoint Filtering** | Allowlist/blocklist URL patterns |
| **Approval Workflows** | Require human approval above a threshold |
| **Storage Backends** | In-memory, JSONL file, or remote API |
| **Dashboard Queries** | Spend by agent, team, endpoint, time range |
